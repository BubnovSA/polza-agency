## Контекст проекта

Ты помогаешь разработать Node.js скрипт для **Polza Agency** — B2B аутрич-агентства.
Задача скрипта: автоматически собрать базу 50+ B2B-компаний, найти контакты, добавить персонализацию через Claude API.

Читай документацию в папке `docs/` перед любыми изменениями:
- `docs/01-pipeline.md` — архитектура пайплайна
- `docs/02-sources.md` — источники данных и как их парсить
- `docs/03-personalization.md` — логика персонализации через Claude API
- `docs/04-email-sequences.md` — цепочки писем (справочник)
- `docs/05-criteria.md` — критерии оценки тестового задания

---

## Стек

- **Runtime**: Node.js 20+, ES Modules (`"type": "module"`)
- **Парсинг**: `cheerio` + `node-fetch`
- **AI**: `@anthropic-ai/sdk` (модель: `claude-haiku-4-5-20251001`)
- **Валидация email**: `dns/promises` (MX-записи)
- **Вывод**: CSV через `csv-writer`, совместимый с Google Sheets
- **Конфиг**: `.env` через `dotenv`

---

## Архитектура — 5 шагов пайплайна

```
[1] HH.ru парсинг вакансий
        ↓ название компании + ссылка на профиль HH + сигнал (название вакансии)
[2] HH профиль компании
        ↓ сайт компании
[3] Сайт компании
        ↓ email (regex) + текст для персонализации
[4] Валидация
        ↓ MX-проверка email
[5] Claude API
        ↓ сигнал + персонализация (1-2 предложения)
        → запись в output/leads.csv
```

## Правила разработки

1. **Graceful errors**: любой fetch в try/catch, скрипт не должен падать на одной компании
2. **Задержки**: минимум 1500ms между запросами к одному домену (не банить)
3. **Логи**: `console.log` с эмодзи для каждого шага (🔍 парсим / ✅ готово / ⚠️ пропускаю)
4. **Комментарии**: каждая функция — JSDoc комментарий
5. **Лимит**: обрабатывать не более 60 компаний за запуск (аргумент CLI)
6. **Fallback**: если email не найден — `info@домен`, пометить флагом `guessed: true`
7. **Дедупликация**: убирать дубли по `company_name` до обработки

## Промпт для Claude (персонализация)

```
Компания: {company_name}
Сигнал: {signal}
Текст с сайта: {site_text_first_1500_chars}

Напиши 1-2 предложения персонализации для холодного B2B письма.
Требования:
- Используй конкретный факт о компании ИЛИ сигнал из HH
- Объясни, почему именно им нужен email-аутрич прямо сейчас
- Никаких общих слов ("ведущая компания", "динамично развивается")
- Только конкретика
Формат: просто текст, без кавычек и markdown.
```

## Частые проблемы и решения

**HH блокирует запросы** → добавь User-Agent, увеличь sleep до 3000ms

**Сайт не открывается** → timeout 8000ms, пропускай с пометкой в логе

**Email не найден на главной** → попробуй `/contacts`, `/about`, `/o-nas`

**Claude возвращает общие слова** → уточни промпт: "запрещено использовать слова: компания, рынок, развитие"

**Селекторы HH сломались** → скажи мне "обнови селекторы HH" и покажи HTML элемента
