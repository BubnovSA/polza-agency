# 04 — Цепочки писем

## Структура цепочки (3 письма)

```
День 0 → Письмо 1: холодное знакомство + запрос контакта
День 3 → Письмо 2: основной оффер + кейс
День 8 → Письмо 3: финальный follow-up + мягкий выход
```

---

## Письмо 1 — Холодное знакомство

**Цель:** найти ЛПР, первый контакт  
**Длина:** 50-80 слов  
**Тема:** сухая, деловая (не рекламная)

### Тема (варианты A/B)
```
A: Вопрос по [Company Name]
B: Подскажите, кто отвечает за продажи в [Company Name]?
```

### Тело
```
{{персонализация}}

Меня зовут [Имя], представляю Polza Agency — помогаем B2B-компаниям 
получать поток входящих заявок через холодные email-рассылки.

Подскажите, кто у вас отвечает за лидогенерацию / развитие продаж?
Хотел бы направить короткое предложение нужному человеку.

[Имя]
```

**Правила:**
- Не представляться полностью (имя без компании) — снижает спам-триггеры
- Не продавать в первом письме — только запросить контакт
- {{персонализация}} — переменная из скрипта (колонка `personalization`)

---

## Письмо 2 — Основной оффер

**Цель:** раскрыть предложение, показать кейс  
**Длина:** 80-120 слов  
**День отправки:** +3 дня после письма 1

### Тема
```
Идея для [Company Name] — поток B2B-лидов
```

### Тело
```
[Имя], добрый день.

Писал вам ранее — хотел поделиться конкретной идеей для [Company Name].

Мы в Polza Agency запускаем email-аутрич для B2B-компаний: пишем 
персонализированные письма потенциальным клиентам и передаём вам 
только заинтересованных лидов.

Кейс: для [похожая компания] за первый месяц получили 12 квалифицированных 
встреч с ЛПР при базе 800 контактов.

Актуально? Готов рассказать подробнее на 20-минутном звонке.

[Имя], Polza Agency
```

---

## Письмо 3 — Финальный follow-up

**Цель:** последнее касание, мягкий выход  
**Длина:** 40-60 слов  
**День отправки:** +5 дней после письма 2

### Тема
```
Re: Idея для [Company Name]
```

### Тело
```
[Имя], последний раз напишу — не хочу надоедать.

Если тема лидогенерации через аутрич сейчас не актуальна — всё окей, 
больше не побеспокою.

Если вдруг появится интерес позже — буду рад пообщаться.

[Имя], Polza Agency
polzaagency.ru
```

---

## Принципы написания (из практики агентства)

1. **Каждое письмо — новая информация.** Не повторять предыдущее
2. **Сигнал → боль → решение.** Структура каждого письма
3. **Кейс важнее описания услуги.** Конкретные цифры убеждают
4. **Короткие абзацы.** Менеджеры читают по диагонали
5. **Один CTA.** Не давать несколько вариантов действия
6. **Тема = деловой вопрос.** Не рекламный заголовок

## Переменные в письмах

| Переменная | Источник | Пример |
|---|---|---|
| {{персонализация}} | Claude API (скрипт) | "Вижу, ищете Head of Sales..." |
| [Company Name] | `company_name` из CSV | "Ромашка CRM" |
| [Имя] | Отправитель | "Никита" |
| [похожая компания] | Кейс агентства | "SaaS-компания из EdTech" |
